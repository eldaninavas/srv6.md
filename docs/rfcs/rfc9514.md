# RFC 9514 - Border Gateway Protocol - Link State (BGP-LS) Extensions for SRv6

| Field | Value |
|-------|-------|
| **Title** | Border Gateway Protocol - Link State (BGP-LS) Extensions for Segment Routing over IPv6 (SRv6) |
| **Authors** | G. Dawra, C. Filsfils, K. Talaulikar, M. Chen, D. Bernier, B. Decraene |
| **Date** | December 2023 |
| **Status** | Proposed Standard |
| **Link** | [datatracker.ietf.org/doc/rfc9514](https://datatracker.ietf.org/doc/rfc9514/) |

## Summary

RFC 9514 defines extensions to the Border Gateway Protocol - Link State (BGP-LS) for carrying SRv6 information. BGP-LS (RFC 7752, updated by RFC 9552) is used to export link-state topology and traffic engineering information from IGP domains to external consumers such as SDN controllers and Path Computation Elements (PCEs). This RFC extends BGP-LS to include SRv6-specific attributes.

The extensions enable a centralized controller to learn the complete SRv6 topology of the network, including SRv6 capabilities, locators, End SIDs, adjacency SIDs (End.X), and the SID structure. With this information, a controller can compute SRv6 paths and program SR Policies without needing to participate in the IGP directly.

This is a critical building block for SDN-based SRv6 deployments where a PCE or SDN controller computes paths centrally and distributes them via PCEP or BGP SR Policy. The BGP-LS SRv6 extensions ensure the controller has a complete and accurate view of the SRv6 data plane.

## Key Sections

| Section | Topic | Description |
|---------|-------|-------------|
| 3 | SRv6 Capabilities | Node-level SRv6 support advertisement via BGP-LS |
| 4 | SRv6 Locator | Prefix NLRI attribute carrying locator and algorithm |
| 5 | SRv6 End SID | BGP-LS attribute TLV for SRv6 endpoint behaviors |
| 6 | SRv6 Adjacency SID (End.X) | BGP-LS link attribute for per-adjacency SRv6 SIDs |

## BGP-LS SRv6 Attribute TLVs

| TLV | Code Point | BGP-LS Attribute | Maps From (IGP) |
|-----|-----------|-----------------|-----------------|
| **SRv6 Capabilities** | 1106 | Node Attribute | IS-IS / OSPFv3 SRv6 Capabilities |
| **SRv6 Locator** | 1162 | Prefix Attribute | IS-IS SRv6 Locator TLV / OSPFv3 SRv6 Locator TLV |
| **SRv6 End SID** | 1163 | Prefix Attribute | IS-IS SRv6 End SID sub-TLV / OSPFv3 SRv6 End SID sub-TLV |
| **SRv6 SID Structure** | 1164 | Prefix/Link Attribute | IS-IS / OSPFv3 SRv6 SID Structure sub-TLV |
| **SRv6 End.X SID** | 1106 | Link Attribute | IS-IS SRv6 End.X SID sub-TLV / OSPFv3 SRv6 Adj SID sub-TLV |

## BGP-LS to IGP Mapping

| BGP-LS Component | IS-IS Source | OSPFv3 Source |
|-----------------|-------------|---------------|
| Node NLRI + SRv6 Caps | Router Capability TLV | Router Information LSA |
| Prefix NLRI + SRv6 Locator | SRv6 Locator TLV | SRv6 Locator TLV in E-Router-LSA |
| Prefix NLRI + SRv6 End SID | SRv6 End SID sub-TLV | SRv6 End SID sub-TLV |
| Link NLRI + SRv6 End.X SID | SRv6 End.X SID sub-TLV | SRv6 Adjacency SID sub-TLV |

## Use with SDN Controllers and PCE

```
+------------------+
|  SDN Controller  |     BGP-LS provides the SRv6 topology database
|  / PCE           |     to the controller, which then computes
+--------+---------+     paths and programs SR Policies via PCEP
         |                or BGP SR Policy.
    BGP-LS Session
         |
+--------+---------+
|   BGP-LS Speaker |     Collects IGP SRv6 data from IS-IS/OSPFv3
|   (Route Reflector)    and exports via BGP-LS to the controller.
+--------+---------+
         |
    IGP Domain (IS-IS or OSPFv3 with SRv6 extensions)
```

| Component | Role |
|-----------|------|
| **BGP-LS Speaker** | Collects SRv6 information from the IGP and exports it northbound |
| **SDN Controller / PCE** | Consumes the BGP-LS SRv6 topology to compute constrained paths |
| **PCEP / BGP SR Policy** | Southbound protocol to push computed SR Policies to headend routers |

!!! warning "Common Pitfalls"
    - BGP-LS SRv6 extensions only export information -- they do not program the data plane. A separate mechanism (PCEP, BGP SR Policy) is needed to install computed paths.
    - The BGP-LS speaker must participate in the IGP to learn SRv6 TLVs. If the IGP peering is misconfigured or the IGP does not carry SRv6 extensions, BGP-LS will have an incomplete view.
    - Ensure the BGP-LS speaker's IGP implementation supports the SRv6-specific TLVs from RFC 9352 (IS-IS) or RFC 9513 (OSPFv3). Older implementations may silently drop unknown TLVs.
    - The SRv6 SID Structure TLV is essential for the controller to correctly parse the LOC:FUNC:ARG boundaries. Missing SID Structure information can lead to incorrect path computation.
    - Multi-domain deployments require BGP-LS from each domain. The controller must stitch cross-domain topology views to compute end-to-end SRv6 paths.
