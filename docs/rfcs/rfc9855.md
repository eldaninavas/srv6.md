# RFC 9855 - TI-LFA

| Field | Value |
|-------|-------|
| **Title** | Topology-Independent Loop-Free Alternate Fast Reroute |
| **Authors** | S. Litkowski, A. Bashandy, C. Filsfils, B. Decraene, P. Francois, D. Voyer |
| **Date** | June 2025 |
| **Status** | Proposed Standard |
| **Link** | [datatracker.ietf.org/doc/rfc9855](https://datatracker.ietf.org/doc/rfc9855/) |

## Summary

RFC 9855 defines Topology-Independent Loop-Free Alternate (TI-LFA), a fast-reroute mechanism for IP and SR networks that provides sub-50ms protection against link, node, and SRLG (Shared Risk Link Group) failures. Unlike traditional LFA (RFC 5286) and Remote LFA (RFC 7490), TI-LFA guarantees 100% coverage for any topology -- hence "topology-independent."

TI-LFA computes backup paths that follow the post-convergence shortest path. When a failure occurs, the protecting node (Point of Local Repair, or PLR) immediately redirects traffic along a pre-computed repair path using a repair segment list. Because the repair path mirrors the post-convergence path, once the IGP reconverges, the traffic is already on the correct path, avoiding sub-optimal routing during the convergence window.

The mechanism relies on Segment Routing to encode the repair path as a short segment list pushed onto the packet at the PLR. This is done using prefix SIDs, adjacency SIDs, or a combination thereof. The key algorithmic concept is the P-node and Q-node (or PQ-node), which represents the point on the post-convergence path where the repair tunnel can safely deliver traffic.

## P-Space, Q-Space, and PQ-Node

| Concept | Definition |
|---------|-----------|
| **P-Space** | Set of nodes reachable from the PLR without traversing the failed element |
| **Q-Space** | Set of nodes that can reach the destination without traversing the failed element (computed on the post-convergence topology) |
| **PQ-Node** | A node in the intersection of P-Space and Q-Space -- the ideal repair tunnel endpoint |
| **P-Node** | A node in P-Space but not Q-Space; requires an additional segment to reach Q-Space |

## Repair Segment List Construction

| Scenario | Repair Segments Needed | Example |
|----------|----------------------|---------|
| **Direct PQ-node exists** | 1 segment (Prefix SID to PQ-node) | PLR pushes {PQ-node Prefix SID} |
| **No direct PQ-node** | 2+ segments (P-node Prefix SID + Adjacency SID to Q-node) | PLR pushes {P-node Prefix SID, Adj-SID to Q-node} |
| **Adjacent PQ-node** | 1 segment (Adjacency SID) | PLR pushes {Adj-SID to neighbor} |

## Protection Types

| Protection Level | What is Protected | Computation |
|-----------------|-------------------|-------------|
| **Link Protection** | Single link failure | Post-convergence SPF excluding the protected link |
| **Node Protection** | Failure of a directly connected neighbor node | Post-convergence SPF excluding the protected node and all its links |
| **SRLG Protection** | All links in a Shared Risk Link Group | Post-convergence SPF excluding all links in the SRLG |

## TI-LFA vs Other FRR Mechanisms

| Mechanism | Coverage | Requires SR | Post-Convergence Optimal | Topology Dependent |
|-----------|----------|-------------|-------------------------|-------------------|
| **LFA (RFC 5286)** | Partial (~40-80%) | No | No | Yes |
| **Remote LFA (RFC 7490)** | Better (~90-95%) | No (uses LDP tunnel) | No | Yes |
| **TI-LFA** | 100% | Yes | Yes | No |

!!! warning "Common Pitfalls"
    - TI-LFA requires Segment Routing to be enabled. Without SR, the repair segment list cannot be constructed.
    - Node protection is strictly stronger than link protection but may produce longer repair segment lists. Some platforms limit the number of repair segments they can push.
    - SRLG protection requires that SRLG membership be consistently configured across all nodes. Inconsistent SRLG definitions lead to inadequate protection.
    - The post-convergence path computation assumes all nodes reconverge to the same topology. Asymmetric link costs or misconfigured metrics can invalidate this assumption.
    - TI-LFA protects the data plane within 50ms, but the control plane still needs full IGP convergence. During the window between FRR activation and convergence, traffic follows the repair path, which may differ from the final converged path.
